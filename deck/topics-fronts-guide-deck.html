PRINCIPLE // 02

A New Front Is Still 3 Things

TELEGRAM

Topic

The conversation surface for one operating area.

WORKSPACE

Folder

The structured operating memory with files, backlog, outputs, reports, data, and scripts.

REGISTRY

Mapping

The fronts registry is the source of truth that connects topic to folder.

FLOW // 03

The Official Client Flow

01

Create the topic

The client opens a new Telegram forum topic.

02

Send the first message

The first message gives the agent an entry point for that unmapped topic.

03

Confirm new front

The agent confirms whether this topic should become a new operating front.

04

Answer bootstrap questions

The agent asks only the missing setup questions.

05

Front created

The system registers the front and confirms it is visible from Mission Control.

QUESTIONS // 04

What The Agent Should Ask

MINIMUM CANVAS

[+]

What should this front be called?

[+]

What is the main objective?

[+]

What kind of outputs should it produce?

[+]

What inputs or sources should it use?

[+]

Are there any constraints, style rules, or boundaries?

IMPORTANT

The user should not need to write all of this in one perfect prompt. The skill should guide the conversation and ask only what is missing.

This keeps front creation natural and makes the system feel like an operating assistant instead of a technical setup ritual.

OUTCOME // 05

What Happens After The Interview

SYSTEM ACTIONS

  • Create the front folder.
  • Create `FRONT.md`, `PROJECT.md`, and `OPERATIONS.md`.
  • Create `outputs/`, `reports/`, `data/`, and `scripts/`.
  • Register the topic-to-folder mapping in `mission-control/fronts.yaml`.

CLIENT CONFIRMATION

  • The agent confirms that the front now exists.
  • The agent confirms that it is visible from Mission Control.
  • The topic can now be used as a normal operating front.

MISTAKES // 06

What To Avoid

DON'T

[!]

Ask the client to use the terminal for normal front creation.

[!]

Expect a perfectly formatted prompt with all fields filled in advance.

[!]

Treat an unmapped topic as a fully operational front before bootstrapping it.

DO

[+]

Use the first message in the new topic as the bootstrap entry point.

[+]

Ask only the missing setup questions.

[+]

Confirm that the front is created and visible in Mission Control.
